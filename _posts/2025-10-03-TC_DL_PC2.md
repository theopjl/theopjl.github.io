### 3. Normalizing the inputs and outputs between [0, 1]

Network training tends to struggle when the input data contain large numbers. That's why we will normalize the inputs.

```python
# GETTING INPUTS / OUTPUTS MIN MAX FROM TRAINING SET FOR NORMALIZATION
x_min_train, x_max_train = X_train.min(axis=0), X_train.max(axis=0)
y_min_train, y_max_train = Y_train.min(axis=0), Y_train.max(axis=0)

def normalize_inputs(X):
    return (X - x_min_train) / (x_max_train - x_min_train)

def unnormalize_inputs(X):
    return X  *(x_max_train - x_min_train) + x_min_train

def normalize_outputs(Y):
    return (Y - y_min_train) / (y_max_train - y_min_train)

def unnormalize_outputs(Y):
    return Y  * (y_max_train - y_min_train) + y_min_train

# NORMALIZATING TRAIN, VAL, TEST INPUTS BETWEEN [0,1]
X_train_norm = normalize_inputs(X_train)
X_val_norm   = normalize_inputs(X_val)
X_test_norm  = normalize_inputs(X_test)

# NORMALIZATING TRAIN, VAL, TEST OUTPUTS BETWEEN [0,1]
Y_train_norm = normalize_outputs(Y_train)
Y_val_norm   = normalize_outputs(Y_val)
Y_test_norm  = normalize_outputs(Y_test)

# CHECKING STANDARD DEVIATION
print("TRAIN Z STD: ", Y_train_norm.std())
```

### Least squares solution
The optimal weights and bias that minimize the L2 loss can be computed in closed form: they correspond to the linear least-squares solution:
$$\begin{pmatrix}w_{11}\\w_{12}\\b_1\end{pmatrix}=(\mathbf{M}^t\mathbf{M})^{-1}\mathbf{M}^t\begin{pmatrix}|\\\mathtt{Y_{elevation}}\\|\end{pmatrix}$$
with $\mathbf{M}=\begin{pmatrix}|&|&|\\\mathtt{X_{lat}}&\mathtt{X_{long}}&1\\|&|&|\end{pmatrix}$

```python
M = np.stack((X_train_norm[:, 0],X_train_norm[:, 1],np.ones_like(X_train_norm[:, 0])),axis=1)
LMS = np.linalg.solve(M.transpose() @ M,M.transpose() @ Y_train_norm)
```

### Backpropagation : Gradient computing

We decided to open the black box of gradients and take a look inside, at the gradient values before and after backpropagation.

```python
def print_gradient(model) :
    for name, p in model.named_parameters() :
        if p.grad is not None :
            print(f" {name}, grad : {p.grad}")
        else:
            print(f"no gradient information!")
    print('\n')


# CONVERTING INPUT AND GROUND TRUTH VALUES OF TRAINING DATA TO TENSORS, TO COMPUTE LOSS
inputs = torch.tensor(X_train_norm, dtype=torch.float32, device=device)
```

### 9.2. Testing

This test function will be used in the validating phase in the training phase.

Down below you'll find 2 more test function used for testing and plotting in 2d or 3d. Theses functions are exactly the same as the *test()*method, except we added graphs to it.

```python
# TEST METHOD
# model     : Model
# loss_fn   : Loss function
# loader : The loader used
# Here, loader is part of the method's parameters. It let you choose the datasets. 
# Because the same function can be used for validating and testing purpose. 
# This parameter prevents for rewriting code, basically we just want to test predictions.
def test(model, loss_fn, loader):

    # PUTTING MODEL IN TESTING MODE
    model.eval()

    # VARIABLES USED FOR PRITTING
    num_batches = len(loader)
    test_loss = 0

    # DEACTIVATING GRADIENT COMPUTE (Here we just want to test, not train. Save precious ressources.)
    with torch.no_grad():
        for X, y in loader:
            # GETTING THE PREDICTION
            pred = model(X)

            # COMPUTING LOSS (Here we are summing to get the average test loss)
            test_loss += loss_fn(pred, y).item()

    test_loss /= num_batches

    # PRITING AVERAGE LOSS
    print("\n---Done testing---")
    print(f"Test Avg loss: {test_loss:.6f}")

    return test_loss
```
